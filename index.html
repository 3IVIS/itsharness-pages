§ 02 · specification

FlowSpec v0.2.0. The IR is frozen. The canvas authors it.

Two required fields, fourteen node types, a discriminated edge union, three optional config sections. The canvas emits this on every change. All four runtime adapters consume it. Below — a real flow, in real JSON.

      "query":    { "type": "string" },

PHASE 4 HIGHLIGHT — ENTERPRISE DELIVERY
The four-runtime NODE_SUPPORT_MATRIX is complete.

The MS Agent Framework adapter (maf_adapter.py, ~800 LOC, semantic-kernel v1.x) closes the matrix. agent_debate maps natively to AgentGroupChat + KernelFunctionTerminationStrategy — the only adapter where this is native rather than synthesised as a loop. Conditional SK imports mean pure transform, HITL, or tool-only flows compile and run without semantic-kernel installed. Alongside MAF: SSO/OIDC + SCIM 2.0, a 5-stage Visual CI/CD pipeline, a 15-file on-prem Helm chart covering all 9 services, Yjs real-time collab with multi-user cursors and IndexedDB offline persistence, and the @itsharness/canvas embeddable npm package.

§ 06 · public alpha

Its Harness is in public alpha. We need your flows.

All four runtimes are executable and the feature set is complete, but this is alpha software — rough edges exist and your real-world flows will find them faster than any test suite. Two things matter most right now: bugs you hit, and features you wish existed.

Public alpha — expect rough edges

APIs may shift, Docker Compose behaviour may differ across environments, and edge cases in less-common node combinations are not yet fully covered. Run it, break it, and tell us what happened. Every report directly shapes what gets fixed next.

track A · run & report

Run it against your flows

One docker compose up starts all nine services. Point it at a real flow — your actual use case, not a toy example. When something breaks, crashes silently, or produces wrong output, open a bug report. Include your FlowSpec JSON, the runtime you used, and the full error. The more specific, the faster it gets fixed.
