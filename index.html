§ 02 · specification

FlowSpec v0.2.0. The IR is live. The canvas authors it.

Two required fields, fourteen node types, a discriminated edge union, three optional config sections. The canvas emits this on every change. Adapters consume it. Below — a real flow, in real JSON.

RFC · last call before adapter codegen
context_from on edges — CrewAI semantics, or generalised to every runtime?

The spec borrows CrewAI's Task.context to thread context across nodes. LangGraph, Mastra and MS Agent Framework would need a shim to honour it. Generalise the semantics in v0.2.1 and let every adapter consume them, or leave it CrewAI-flavoured and have the other adapters translate? Working assumption: generalise. Real-world voices wanted before we hardcode it into the adapter.

§ 06 · getting involved

The design is open. The decisions are still cheap.

We're deliberately slow before we're fast. The IR, the node taxonomy, and the adapter contract land first — code follows. Three lanes, three weeks-of-effort, pick whichever fits.

The neutral IR is the lingua franca between canvas and four adapters. Every shape decision ripples through all of them. Read the v0.1 spec, push back, propose alternatives.
