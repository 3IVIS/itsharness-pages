§ 02 · specification

FlowSpec v0.1.0. The IR is real. The shape is settled.

Two required fields, twelve node types, a discriminated edge union, three optional config sections. Adapters consume it. The canvas authors it. Humans review it. Below — a real flow, in real JSON.

Q35 · multi_agent_chat — should LG/MA approximate Microsoft's AgentGroupChat, or refuse to compile?

MS AF's group chat is fundamentally conversational. LangGraph and Mastra use hierarchical graph nesting — structurally different execution models. Approximate with a warning, or refuse to compile and break the "write once, run anywhere" promise? Working assumption: approximate with warning. Real-world voices wanted before we lock it in.

§ 06 · getting involved

The design is open. The decisions are still cheap.

We're deliberately slow before we're fast. The IR, the node taxonomy, and the adapter contract land first — code follows. Three lanes, three weeks-of-effort, pick whichever fits.
